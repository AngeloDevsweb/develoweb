NODE JS

Node js es un entorno de ejecución de javascript de lado del servidor que usa un modelo asíncrono y que está dirigido por eventos.

Node js fue creado por los mismos desarrolladores de javascript, quienes modificaron todo para que no solo pueda ejecutarse en el navegador sino también en el lado del servidor, como si de aplicaciones independientes se tratara.

¿Cómo funciona node js?

El funcionamiento que tiene por dentro este entorno de javascript es bastante interesante ya que el nodo está controlado completamente por eventos. Es decir, que uno de los factores importantes que tiene node es que puede tener muchas conexiones abiertas y esperando.

Node js opera en un solo subproceso utilizando un modelo entrada y entrada sin bloqueo de salida, lo que permite soportar muchas conexiones al mismo tiempo mantenidas en el bucle de eventos. A esto se lo denomina también modelo asíncrono y en términos sencillos a manera de darse una idea es como hacer una petición al servidor y este la procese, pero si le toma más tiempo el completarla y se hace una segunda petición al servidor, este no la bloqueara, lanzara un evento o una respuesta de la segunda solicitud mientras termina el proceso para responder a la primera y asi sucesivamente.

Bueno esa sería una forma de tratar de explicarlo ya que por dentro el proceso es un poco más complejo de entender. Pero asi se puede tener una idea de cómo funciona node js.

Ademas de esto node js, se ha convertido en una tecnología muy importante ya que gracias a ella, nosotros podemos ejecutar javascript en el lado del servidor, cosa que antes era imposible. Primero porque antiguamente se utilizaba php para poder crear una app web completa, eso usando html, css , javascript para el cliente y para el lado del servidor php y mysql. Por lo general eran estas tecnologías las que se usaban para desarrollar una página web dinámica.

Pero todo eso cambio gracias a node js, porque su implementación llevo a que hoy javascript pueda ejecutarse del lado del servidor, dando como resultado una facilidad para los desarrolladores de poder utilizar una sola tecnología tanto en cliente como en servidor. Lo que hizo que también se puedan trabajar por separado, una independientemente de la otra.

Si estas desarrollando frontend, tan solo tendrás que preocuparte por maquetar tu sitio y realizar las peticiones al servidor, lo mismo con el backend tan solo tendrás que preocuparte de que todo esté bien estructurado para que el servidor pueda dar una respuesta al cliente.
